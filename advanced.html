Section A · ~2 hours

Prompt Caching

Session 6 showed you the basics. Now measure and maximize it.

Quick win · 5 min

Establish your baseline

Run the pre-written warm-up in caching.py: three calls with a large system prompt, printing the cache-creation and cache-read tokens for each. Run 1 creates the cache; runs 2–3 read it, far cheaper. This whole section is about getting run 2's numbers more often.

Warm-up steps → caching.py

A1 What & where to cache 30 min

System prompts aren't the only cacheable thing — large documents and tool definitions are too. cache_control is a watermark: everything before it in token order gets cached. Where you place it decides how much you save.

Walkthrough & tasks → caching.py · Section A1

A2 Agent-loop caching 20 min

In a multi-step loop the system prompt and tools are identical every iteration. With caching, after step 1 you only pay for the new messages — and the payoff compounds as the loop runs longer.

Walkthrough & tasks → caching.py · Section A2

No mini project — your A2 run_agent_loop numbers are the deliverable: cached vs uncached token totals, cache-hit percentage, and the break-even step count. What to record → caching.py · Deliverable

Section B · ~2.5 hours

Advanced Agentic Patterns

Four patterns that go beyond ReAct, plan-execute, and multi-agent.

Quick win · 10 min

Add reflection in 3 lines

Take any prior agent response and add one more call that critiques it. That's reflection — a single extra call that measurably improves output. The rest of this section builds on it.

Warm-up steps → patterns.py

B1 Reflection loop 20 min

Draft → critique → revise, repeating until the model rates its own output "good" or hits a max-iteration cap. The stopping criterion is what keeps it from looping forever.

Walkthrough & tasks → patterns.py · Section B1

B2 Parallel fan-out 25 min

Fork work to N independent subagents in parallel threads, each with its own context, then merge with one synthesis call. The real speedup is rarely exactly N×.

Walkthrough & tasks → patterns.py · Section B2

B3 Router pattern 20 min

A fast, cheap model classifies the request and routes it to a specialist — doing no domain work itself. The craft is fixing one category's misroutes without creating new ones elsewhere.

Walkthrough & tasks → patterns.py · Section B3

B4 Human-in-the-loop 15 min

An ask_human(question) tool pauses the loop for real input() and feeds the answer back as a tool_result. Getting it to reliably pause before acting is a prompt-phrasing problem.

Walkthrough & tasks → patterns.py · Section B4

No mini project — the four pattern implementations in patterns.py are the deliverable, along with their outputs (reflection logs, fan-out timing, router misroutes, HITL transcript). What to keep → patterns.py

Section C · ~1.5 hours

Subagents vs Skills

Learn when a function is enough, when a class is better, and when you actually need a Claude call.

Quick win · 10 min

Cost comparison in real numbers

The warm-up in skills.py solves one task two ways — a direct Python function (0 tokens, <1ms) and a Claude subagent (~50 tokens, ~1s). The numbers make the tradeoff obvious: only spend a subagent when you actually need language understanding.

Warm-up steps → skills.py

C1 The decision framework 15 min

Four levels of capability: tool (pure Python), skill module (a class wrapping tools), subagent (a separate Claude call), and independent agent (a full loop). Some real capabilities genuinely sit on the boundary.

Walkthrough & tasks → skills.py · Section C1

C2 Skill modules & specialization 35 min

A skill is a class bundling related tools, schemas, and error handling. Build MemorySkill alongside the provided SearchSkill/FileSkill, then give specialized agents only the skills they need so each one's context stays small.

Walkthrough & tasks → skills.py · Section C2

No mini project — the C2 two-phase pipeline is the deliverable. The challenge asks you to swap SearchSkill's internals without touching ResearcherAgent: if it still runs, the interface held. Swap test → skills.py · Section C2 Challenge

Section D · ~2 hours

Plugins & MCP

Decouple your tools from your agent code with the Model Context Protocol.

Quick win · 10 min

Hello-world MCP server

Install FastMCP and run the pre-written server in test mode — hello, write_file, and read_file respond locally. The tools look just like Session 2's; MCP only standardizes how they're discovered and called.

Warm-up steps → mcp.py

D1 Tools via MCP 20 min

MCP tools are decorated Python functions; @mcp.tool() exposes them over JSON-RPC and clients discover them automatically. You'll add one and call it live from Claude Code.

Walkthrough & tasks → mcp.py · Section D1

D2 Resources & prompts 30 min

Resources are read-only data the model fetches (vs. tools, which are actions); prompts are reusable parameterized templates any client can invoke. Both move shared context out of inline strings and into the server.

Walkthrough & tasks → mcp.py · Section D2

No mini project — add retrieve_context (cosine search from Session 5) as an MCP tool and a --test branch, then call it from Claude Code via /mcp. That end-to-end path is the deliverable. What to build → mcp.py · Deliverable

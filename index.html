1A Read the response & count tokens 30 min

A response is more than reply text — it carries stop_reason, usage, and a content list. And cost, context limits, and speed are all measured in tokens, not words or characters.

1B Statelessness — and the fix 30 min

Every API call is independent — the model has no memory between calls. The fix is to resend the whole conversation each time: history lives in your code, not the API. Every "agent memory" technique is a workaround for this one fact.

1C System prompt as a dial 15 min

The system prompt is the highest-authority instruction in every call. It shapes tone and format downstream — and for an application, consistency of format matters more than raw quality.

Session 2 · ~2.5 hours

Give the Model Hands

Turn a chatbot into something that acts.

Quick win · 10 min

Define a tool — see what comes back

Add a tools parameter and ask the model to use it. Notice what's missing: an answer. The model returns a request to call a function — it declares intent. That distinction is the whole session.

Warm-up steps → session2.py

2A tool_use → close the loop 35 min

The content list holds a ToolUseBlock with an id, name, and input. You run the function and report back with a matching tool_result. That cycle — tool_use → execute → tool_result — is the mechanical core of every agent ever built.

Walkthrough & tasks → session2.py · Chunk 2A

2B The agent loop 30 min

Wrap the tool cycle in a while loop that keeps running until the model signals it's done (stop_reason == "end_turn"). That repeating heartbeat is what turns a single call into an agent.

Walkthrough & tasks → session2.py · Chunk 2B

2C Failure modes & safety valves 25 min

Make a tool always fail and watch how the model copes (give up? loop? fabricate?). Then add hard limits — MAX_STEPS and a token budget — so a loop can never run forever or surprise you on cost.

Session 3 · ~2.5 hours

Context is Your Budget

Hit the limits, then build instincts to manage them.

Quick win · 10 min

Watch the tokens climb

Log r.usage.input_tokens at the top of each loop iteration and run a 15-step task. The number grows every step — even when nothing new happens. That's your context window filling up, and everything this session answers that problem.

Warm-up steps → session3.py

3A Wire a real search tool 25 min

Swap your fake search for the real Brave Search API (free tier) so the agent can reach current knowledge. The wording of the tool description visibly changes how often the model reaches for it.

Walkthrough & tasks → session3.py · Chunk 3A

3B Compression & the cost of forgetting 30 min

When history gets too large, summarize the oldest turns and drop them — trading precision for space. The cost is real: a fact seen early may be recalled, admitted-unknown, or confabulated later. There's no fix, only a design decision you make on purpose.

Walkthrough & tasks → session3.py · Chunk 3B

3C Structured logging 15 min

When a 10-step loop fails on step 7, a structured log is the only way to reconstruct why. Add it now, not later — every decision, tool call, and token count.

Session 4 · ~3 hours

Architecture Patterns

There are several ways to structure an agent. Each one wins in different situations.

Quick win · 10 min

Turn on Thought lines

Add one sentence to your system prompt asking the model to think out loud before each action, then run a Session 3 task. That single line is the ReAct pattern — watch its stated reasoning against the tool it actually picks.

Warm-up steps → session4.py

4A ReAct — thinking before acting 20 min

Reason + Act: the model states a Thought: before each action. Better decisions, plus a trace you can debug — though the stated reasoning and the chosen tool don't always match.

Walkthrough & tasks → session4.py · Chunk 4A

4B Plan-and-execute — and break it 25 min

Plan first, then execute the steps. Great for well-defined tasks, fragile when reality diverges mid-run — which you'll deliberately trigger with a task whose later steps depend on earlier results.

Walkthrough & tasks → session4.py · Chunk 4B

4C Multi-agent — orchestrator & sub-agent 30 min

An orchestrator whose only tool is call_subagent(task), delegating to your Session 2 loop. The key insight: each agent gets its own independent context window, so sub-agent contexts stay short.

Walkthrough & tasks → session4.py · Chunk 4C

No mini project — instead, run one fixed task through all three patterns and compare steps, tokens, and quality. The comparison table is your deliverable. Exact task & method → session4.py · Compare

Session 5 · ~2.5 hours

Memory & Retrieval

Give your agent knowledge that doesn't fit in the context window.

Quick win · 10 min

Feel semantic similarity

Install sentence-transformers, embed two sentences, and compare their similarity. Same idea → similar vectors; different topic → very different vectors. That gap is semantic search in action.

Warm-up steps → session5.py

5A Cosine similarity & semantic search 30 min

A raw dot product favors longer vectors regardless of meaning; cosine similarity normalizes that out. Implement it yourself, then retrieve the right chunk for a query whose words don't appear in it — that's semantic search.

Walkthrough & tasks → session5.py · Chunk 5A

5B RAG as a tool — and its edges 35 min

Wire retrieval in as a tool the agent chooses to call. Then push the edges: a paraphrased question, a question with no answer in the base, and an ambiguous one that two chunks match — and watch when it answers, declines, or makes something up.

Session 6 · ~2 hours

Production Basics

Make it real — reliability, cost, and responsiveness.

Quick win · 10 min

Watch it die

Wrap your API call so the first attempt raises an error, then run the agent. It crashes with no recovery — that's your current production reliability. This session is the fix.

Warm-up steps → session6.py

6A Retry with exponential backoff 20 min

Retry transient errors (rate limits, timeouts, 5xx); don't retry permanent ones (bad key, malformed request). Simulating failure rates shows exactly how much error budget retries actually buy you.

Walkthrough & tasks → session6.py · Chunk 6A

6B Streaming & structured output 30 min

Streaming delivers tokens as they're generated — same total time, far better perceived responsiveness. Pair it with a required JSON summary and a self-correction retry when the model doesn't comply.

Walkthrough & tasks → session6.py · Chunk 6B

6C Prompt caching 20 min

Your system prompt and tool definitions are identical on every call, yet you pay full price each time. Caching cuts that to ~10% — run the same task twice and measure the cache hits on run 2.

Walkthrough & tasks → session6.py · Chunk 6C

No mini project — your before/after numbers are the deliverable: retry recovery rate, cache-hit percentage, and malformed-output rate with and without format instructions. What to log → session6.py
